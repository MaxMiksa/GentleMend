# PRD — 浅愈(GentleMend) 乳腺癌副作用智能评估系统

## 1. 产品定位

### 1.1 一句话描述
为乳腺癌患者提供副作用自评服务的智能评估系统，通过规则引擎+AI增强实现风险分层，输出可解释的处置建议。

### 1.2 目标用户
- **主要用户**：乳腺癌治疗中/治疗后的患者
- **次要用户**：患者管理团队（查看评估记录、响应协同请求）

### 1.3 业务上下文
该系统是专科医患数据链路的一个关键触点——患者端的副作用上报入口。它连接的是：
- 上游：患者的治疗方案、用药记录（本MVP中简化为用户自述）
- 下游：医疗团队的协同响应、患者病史管理系统

### 1.4 核心价值
| 对患者 | 对医疗团队 |
|--------|-----------|
| 即时获得副作用风险评估 | 高风险患者自动预警，减少漏诊 |
| 清晰的下一步行动指引 | 结构化的副作用数据沉淀 |
| 降低焦虑（低风险时给予安心） | 可审计的评估记录链 |

## 2. 功能需求

### 2.1 前端页面

#### P0 — 用户输入页
- 文本输入区：用户描述当前副作用症状
- 可选的结构化辅助输入（症状类型选择、严重程度滑块）
- 提交按钮触发评估

#### P0 — 结果页
- 风险等级展示（高/中/低，视觉区分）
- 下一步建议（具体行动指引）
- 是否建议联系团队（高风险时突出显示联系入口）
- 依据说明（命中了哪条规则、规则版本号）
- 评估时间戳

#### P0 — 历史记录页
- 按时间倒序展示历史评估记录
- 每条记录可展开查看完整详情
- 风险等级标签筛选

### 2.2 后端接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/assessments` | POST | 提交副作用描述，触发评估 |
| `/api/assessments/:id` | GET | 获取单次评估结果 |
| `/api/assessments` | GET | 获取历史评估列表 |
| `/api/contact-requests` | POST | 创建协同请求（联系团队） |

### 2.3 风险分层规则

三级风险分层，规则引擎作为确定性底线：

| 等级 | 触发条件（示例） | 处置建议 |
|------|-----------------|---------|
| 高风险 | 呼吸困难、严重过敏反应、持续高热(>39°C)、严重出血 | 立即线下就医 / 24h内联系团队 |
| 中风险 | 持续呕吐、中度疼痛、皮疹扩散、轻度感染征象 | 联系团队或密切观察 |
| 低风险 | 轻度疲劳、轻微恶心、脱发、轻度皮肤干燥 | 继续观察与记录 |

规则要求：
- 每条规则有唯一ID、版本号、生效时间
- 规则命中记录必须持久化
- 支持多条规则同时命中时取最高风险等级

### 2.4 数据存储要求

必须包含以下数据实体：
- **Assessment** — 评估主记录（输入、风险等级、时间戳）
- **Advice** — 生成的建议内容
- **Evidence** — 依据说明（命中规则、置信度）
- **RuleSource** — 规则定义（ID、版本、条件、建议模板）
- **EventLog** — 可观测性事件日志

### 2.5 可观测性事件

必须埋入的5个事件：

| 事件名 | 触发时机 | 携带数据 |
|--------|---------|---------|
| `assessment_started` | 用户开始填写 | session_id, timestamp |
| `assessment_submitted` | 用户提交评估 | assessment_id, input_length |
| `result_viewed` | 用户查看结果 | assessment_id, risk_level |
| `contact_team_clicked` | 用户点击联系团队 | assessment_id, risk_level |
| `assessment_closed` | 用户关闭/离开结果页 | assessment_id, duration |

### 2.6 审计要求

每次评估结果必须记录：
- 命中的规则ID列表及各规则版本号
- 结果生成时间（精确到毫秒）
- 规则引擎版本号
- 若使用AI增强，记录模型版本、prompt版本、原始输出

## 3. 非功能需求

### 3.1 安全性
- 评估数据不可变（immutable），生成后只能追加新版本
- 敏感数据传输加密
- API鉴权（MVP阶段可简化为token）

### 3.2 可扩展性
- 规则引擎与业务逻辑解耦，支持规则热更新
- AI增强层可插拔，不影响规则引擎基线功能
- 事件系统支持扩展新事件类型

### 3.3 可解释性
- 每条建议必须关联到具体规则来源
- AI生成的内容必须标注为AI辅助生成
- 用户可查看完整的评估依据链

## 4. 智能体闭环要求

系统需体现"感知-决策-执行-学习"闭环：

- **感知**：接收并理解用户的自然语言副作用描述
- **决策**：规则引擎+AI增强的风险分层判断
- **执行**：生成结构化的评估结果和行动建议
- **学习**：基于历史数据和反馈优化评估准确性

## 5. MVP边界

### 包含
- 完整的三页前端（输入/结果/历史）
- 四个核心API接口
- 规则引擎风险分层（至少覆盖常见乳腺癌治疗副作用）
- 完整的审计链
- 5个可观测性事件
- 智能体闭环架构设计（学习环节可为架构预留，不要求完整实现）

### 不包含
- 用户注册/登录体系（MVP用固定用户）
- 医疗团队端管理后台
- 真实的AI模型训练
- 多语言支持
- 移动端适配
